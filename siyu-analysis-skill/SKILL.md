## 一、工作流概述

本 Skill 基于品牌私域运营数据底表，通过 **4层递进分析模型**（效果验证 -> 对比验证 -> 归因验证 -> 放大验证），生成标准化的私域运营效果分析报告。

分析核心问题：**私域运营是否真正提升了用户消费频次？提升的幅度和归因是否可靠？**

### 1.1 四层分析模型总览

| 层级 | 模型名称 | 分析目的 | 核心问题 |
|:---:|:---|:---|:---|
| L1 | 模型一：效果验证 | 存量老客户在私域运营后的频次变化 | 私域运营后，老客户是否保持活跃并沉淀为高频用户？ |
| L2 | 模型二：对比验证 | 私域组 vs 非私域对照组的大盘差异 | 私域用户的消费频次是否显著高于非私域用户？ |
| L3 | 模型三：归因验证 | 私域首单新客 vs 非私域首单新客的激活差异 | 私域运营是否有效激活了新客的后续消费？ |
| L4 | 模型四：放大验证 | S卡权益对私域用户的消费放大效应 | S卡会员权益是否进一步放大了私域用户的消费频次？ |

四层模型呈递进关系：L1验证私域对存量用户的效果，L2通过对照组排除自选择偏差，L3聚焦新客归因，L4探索权益放大效应。

### 1.2 核心结论维度

每个品牌报告必须围绕以下4个维度给出结论：

1. **高价值沉淀**（模型一）：存量老客户进入私域后的频次变化与高频用户沉淀情况
2. **大盘差异**（模型二）：私域组与非私域组的消费频次差异及趋势
3. **新客激活**（模型三）：私域首单新客 vs 非私域首单新客的后续激活率差异
4. **S卡放大**（模型四）：S卡会员对私域用户消费频次的放大效应

## 二、完整工作流程

### Step 1: 读取Excel数据底表

**输入文件要求：**
- 格式：`.xlsx`
- 必须包含4个sheet，分别命名为：`模型一`、`模型二`、`模型三`、`模型四`
- 每个sheet包含该模型所需的原始数据表

**读取后执行数据清洗：**
- 检查每个sheet的列名是否完整
- 识别缺失值并标记
- 统一数据类型（日期、数值、字符串）
- 验证关键字段：会员ID、消费时间、消费次数、是否私域、是否S卡等

### Step 2: 确认业务背景

在执行分析前，必须确认以下业务背景信息：

| 背景项 | 说明 | 示例 |
|:---|:---|:---|
| 品类 | 品牌所属餐饮品类 | 中式正餐、快餐、茶饮等 |
| 人均消费 | 客单价（元） | 85元 |
| 淡旺季规律 | 品牌自身的季节性波动 | 夏季为旺季，冬季为淡季 |
| S卡价格权益 | 年费会员卡的定价与核心权益 | 199元/年，含生日券、折扣等 |

**业务背景的作用：**
- 解释数据波动（如淡旺季导致的频次自然变化）
- 判断消费频次是否合理（人均85元的品牌，月均2次与月均4次的意义不同）
- 评估S卡权益吸引力

### Step 3: 执行4层分析模型

#### 3.1 模型一：效果验证（高价值沉淀）

**分析目的：** 看存量老客户在私域运营后的频次变化，判断是否保持活跃并沉淀高频用户。

**筛选口径：**
- **样本人群**：指定时间段内进入企微私域的用户
- **时间锚点**：以每位用户进入私域时间为锚点，统计追踪窗口内的消费数据
- **剔除规则①**：加入私域后同时完成首单的新用户（因"加私域下单有礼"会造成0→1的噪音数据，仅保留首单时间不在拉新时间段的用户）
- **剔除规则②**：私域运营前后均无消费记录的僵尸用户（稀释0次占比）

**计算指标：**
- 基准窗口平均消费次数（进入私域前）
- 追踪后1期、2期、3期的平均消费次数
- 高频用户（≥3次/期）占比变化
- 0次消费用户占比变化

**输出图表：**
1. 基准 vs 追踪后各期平均消费次数对比（柱状图）
2. 消费频次分布变化（堆叠柱状图或分组柱状图）

**核心发现写作规范：**
- 先看总体趋势：基准到追踪后是上升、下降还是持平
- 再看结构变化：高频用户占比是否提升，0次用户是否减少
- 最后结合淡旺季做客观判断：下降是否因淡季，上升是否因旺季

**口径注意：**
- 本模型频次变化需结合品牌淡旺季做客观分析
- 剔除首单和僵尸用户后，样本量可能较小，结论适用于存量老客户的活跃维持判断

#### 3.2 模型二：对比验证（大盘差异）

**分析目的：** 通过私域组 vs 非私域对照组，排除自选择偏差，验证私域运营的真实效果。

**筛选口径：**
- **样本人群**：指定时间段内带有会员ID的消费用户（不含平台买单无法识别会员ID的用户）
- **分组**：
  - 私域组：加企微私域的会员
  - 非私域对照组：品牌会员但不在私域中
- **时间锚点**：以用户在筛选时间段内的最早消费时间为锚点，统计追踪窗口内的消费数据

**计算指标：**
- 两组各期的平均消费次数
- 两组高频用户占比
- 两组0次消费用户占比
- 差异趋势（逐期扩大、收敛或稳定）

**输出图表：**
1. 私域组 vs 非私域组平均消费次数趋势（折线图/分组柱状图）
2. 两组消费频次分布对比（分组柱状图）

**核心发现写作规范：**
- 先看两组差异方向：私域组是否高于非私域组
- 再看差异趋势：差异是扩大、收敛还是稳定
- 模型二不做前后对比，重点看两组差异在追踪期间是否稳定或扩大
- 图4（两组平均消费次数趋势对比）仅显示追踪期间数据，不显示初始基准

**口径注意：**
- 模型二不做前后对比，重点看两组差异在追踪期间是否稳定或扩大
- 图4仅显示追踪期间数据，不显示初始基准（因初始基准包含0→1新客，不纳入前后对比）

#### 3.3 模型三：归因验证（新客激活）

**分析目的：** 验证私域运营是否有效激活了新客的后续消费。

**筛选口径：**
- **样本人群**：指定时间段内带有会员ID的消费用户（不含平台买单无法识别会员ID的用户）
- **分组**：
  - 私域首单新客：首次消费且当天加入企微私域的用户
  - 非私域首单新客：完成首次消费但不在私域中的用户
- **时间锚点**：以用户在筛选时间段内的最早消费时间为锚点，统计追踪窗口内的消费数据

**计算指标：**
- 两组各期的平均消费次数
- 两组复购率（追踪期内消费≥2次的占比）
- 两组0次消费用户占比
- 差异趋势

**输出图表：**
1. 私域首单新客 vs 非私域首单新客平均消费次数趋势
2. 两组复购率对比

**核心发现写作规范：**
- 两组样本均为首单新客，基线一致，差异可直接归因于私域运营效应
- 重点看：私域首单新客的复购率是否显著高于非私域首单新客
- 如果差异不明显，需客观说明，不硬写价值

**口径注意：**
- 本模型两组样本均为首单新客，基线一致，差异可直接归因于私域运营效应

#### 3.4 模型四：放大验证（S卡放大）

**分析目的：** 验证S卡会员权益是否进一步放大了私域用户的消费频次。

**筛选口径：**
- **样本人群**：私域用户中已购买S卡和未购买S卡的用户
- **分组**：
  - S卡私域用户：私域用户且购买了S卡会员
  - 非S卡私域用户：私域用户但未购买S卡会员
- **时间锚点**：以用户进入私域时间为锚点，统计追踪窗口内的消费数据

**计算指标：**
- 两组各期的平均消费次数
- S卡购买率（私域用户中购买S卡的比例）
- S卡带来的频次提升幅度
- ROI估算（S卡年费 vs 额外消费带来的收入）

**输出图表：**
1. S卡私域用户 vs 非S卡私域用户平均消费次数趋势
2. S卡购买率与频次提升关系

**核心发现写作规范：**
- 先看S卡是否带来频次提升
- 再看提升幅度是否值得年费投入
- 注意：S卡用户可能是本身消费意愿更强的用户，存在自选择偏差

**口径注意：**
- S卡用户可能存在自选择偏差（本身高消费意愿的用户更可能购买S卡）
- ROI估算需保守，不做过度推断
- 模型四取数逻辑与模型二一致，初始基准包含0→1新客，不做前后对比
- 图8（购S卡 vs 未购S卡趋势图）仅显示追踪期间数据，不显示初始基准

### Step 7: 结论校验

在报告最终输出前，必须执行以下4项校验：

#### 校验清单

| 校验项 | 检查内容 | 通过标准 |
|:---|:---|:---|
| **数据事实** | 所有数字是否与原始数据一致 | 抽样核对5个以上关键数字，100%准确 |
| **结论依据** | 每个结论性表述是否有数据支撑 | 每条结论都能找到对应的数据表格或图表 |
| **口径一致** | 全文口径是否统一 | "私域首单新客"和"非私域首单新客"的定义在全文中一致 |
| **内容平衡** | 是否不硬找问题也不硬写价值 | 负面数据有客观说明，正面数据有口径限定，无明显 bias |

**校验失败处理：**
- 数据事实不符：返回Step 3重新核对数据源
- 结论无依据：补充数据或删除该结论
- 口径不一致：统一全文术语
- 内容不平衡：调整表述语气，增加限定词

## 三、报告结构（V2 MECE优化版）

报告HTML模板采用以下8节结构：

### 3.1 报告头部（Header）

包含元素：
- 品牌名称 + "私域运营效果分析报告"
- 数据时间段、追踪窗口、对比周期
- 业务背景条（品类 | 人均消费 | 淡旺季）
- 全局口径声明（平均消费次数的计算方式、S卡定义）

### 3.2 执行摘要（Executive Summary）

**结构（MECE优化）：**
- 核心结论汇总表（4行 × 3列：结论维度 | 判断 | 依据）
- KPI看板（3张关键指标卡片）
- 数据说明/异常信号（如有）

**核心结论汇总表：**

| 结论维度 | 判断 | 依据 |
|:---|:---|:---|
| 高价值沉淀 | ✅ 有效 / ❌ 不明显 / ⚠️ 需观察 | 具体数据 |
| 大盘差异 | ✅ 显著 / ❌ 不显著 / ⚠️ 待验证 | 具体数据 |
| 新客激活 | ✅ 有效 / ❌ 不明显 / ⚠️ 需观察 | 具体数据 |
| S卡放大 | ✅ 显著 / ❌ 不显著 / ⚠️ 样本不足 | 具体数据 |

判断标签样式：
- `yes`（绿色）—— 数据支持正面结论
- `no`（红色）—— 数据不支持正面结论
- `maybe`（琥珀色）—— 数据有限，需观察

### 3.3 模型分析层

**每个模型的统一内部顺序（5部分）：**

```
模型block
├── 模型头部（名称 + 叙事标签）
├── 模型说明（一句话说明分析目的）
├── 方法口径（methodology-box）
│   └── 筛选口径列表（样本人群、时间锚点、剔除规则）
├── 数据可视化（图表）
│   └── 1-2张ECharts图表
├── 数据明细表（table）
│   └── 支持rowspan + 差异趋势列
├── 核心发现（Callout框，positive/neutral/risk）
└── 口径注意（Callout框，warning）
```

**叙事标签（model-tag）：**
- 模型一：效果验证
- 模型二：对比验证
- 模型三：归因验证
- 模型四：放大验证

### 3.4 交叉对比与归因

- 综合多个模型的数据，发现跨模型的关联规律
- 示例：模型一显示存量用户频次提升 + 模型二显示私域组高于非私域组 = 私域运营对存量和增量均有正向作用

### 3.5 数据攻防推演

**结构：** 质疑 + 回应

模拟品牌方视角对数据结论的典型质疑：
1. "私域组消费频次高，是不是因为本身高频用户更愿意加私域？"
2. "追踪窗口内的下降是不是因为淡季？"
3. "样本量这么小，结论可靠吗？"
4. "S卡用户消费高，是不是因为他们本来就更爱消费？"

每条质疑配数据回应，增强报告可信度。

### 3.6 运营建议

**结构：**
- 每条建议包含：优先级（P0/P1/P2） + 时间属性 + 数据来源模型 + 具体动作 + 量化预期效果
- 取消短/中/长时间分组（边界模糊）
- 建议自带时间属性（如"7日内"、"下月"、"本季度"）

**优先级定义：**
- **P0**（红色）：紧急且重要，需立即执行
- **P1**（琥珀色）：重要但不紧急，纳入计划
- **P2**（灰色）：优化项，条件允许时执行

### 3.7 最终结论

- 用1-3句话总结全报告核心结论
- 放在独立的 `final-conclusion` section 中
- 使用 neutral Callout 样式

### 3.8 附录

包含4个子项：
- **A. 模型定义与筛选口径**：三个模型的详细定义 recap
- **B. 数据口径说明**：追踪窗口、对比周期、消费次数计算、客户分层标准
- **C. 术语表**：私域客户、非私域对照组、高频用户等
- **D. 局限性/偏差说明**：自选择偏差、季节性因素、样本局限

## 五、关键原则

### 5.1 结论客观性

- **不硬找问题**：如果数据整体向好，不人为挖掘负面
- **不硬写价值**：如果数据不支持正面结论，不强行包装
- **每个结论必须有数据支撑**：任何判断性语句后必须紧跟数据

### 5.2 推断边界

- **不做无数据支撑的跨季节推断**：
  - 正确："4月进入私域的用户在5-6月频次下降，与该品牌夏季为淡季的规律一致"
  - 错误："预计7-8月频次会继续下降"（如无7-8月数据）
- **不做无数据支撑的跨品牌对比**：
  - 正确："本品牌私域用户月均消费2.3次"
  - 错误："本品牌私域活跃度高于行业平均水平"（如无行业数据）

### 5.3 分组口径统一

全文中以下分组术语必须严格统一：

| 术语 | 定义 |
|:---|:---|
| 私域首单新客 | 首次消费且当天加入企微私域的用户 |
| 非私域首单新客 | 完成首次消费但不在私域中的用户 |
| 私域组 | 加企微私域的品牌会员 |
| 非私域对照组 | 品牌会员但不在企微私域中 |
| S卡私域用户 | 私域用户中购买了S卡会员的用户 |
| 非S卡私域用户 | 私域用户中未购买S卡会员的用户 |

### 5.4 数字规范

- 消费次数保留1位小数（如 2.3次）
- 百分比保留1位小数（如 34.5%）
- 金额保留2位小数（如 85.00元）
- 样本量使用整数（如 1,234人）
